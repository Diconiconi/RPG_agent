## 当前方向
- 用户决定不再另起一个自有 mod 工程，而是直接基于 `参考mod` 改善现有 AI 对话 mod。
- 工作区应以 `参考mod` 为主体；后续开发优先研究并改造 `参考mod/plugins/MCS_AIChatMod.dll` 及其周边配置。
- 不要修改真实游戏安装目录。`F:\SteamLibrary\steamapps\common\觅长生` 默认只读。
- 不要提交 API key、认证文件、游戏存档、运行日志、真实游戏文件或复制来的二进制依赖。

## 已确认的设计决策
- 第一版目标：通过 AI 对话真实修改游戏内 NPC 好感度/关系，不再写入其他关系数值通道。
- 关系变化方式：AI 只输出“建议效果”，Mod 负责校验和执行。
- 玩家反馈：游戏内显示简短提示，例如“对方态度略有缓和：好感度 +2”。
- 单轮变化范围：按事件分档校验；通过聊天获得的正向好感度最高到 `60`，达到或超过 `60` 后不再通过聊天增加好感度。
- 负向对话效果不受 `60` 上限限制，辱骂、威胁、背叛等仍可降低好感度。
- 对话触发战斗 V2：已有好感度分级不变，只有严重冲突且 NPC 正文明确宣战时，才在好感度扣减后触发标准战斗。

## 推荐机制草案
每轮玩家发言后，AI 正常生成 NPC 回复，同时附带一段隐藏结构化效果建议：

```json
{
  "favor_delta": 2,
  "reason": "玩家语气尊重，并主动表达善意",
  "memory_summary": "玩家曾以礼相待，使该 NPC 态度稍有缓和"
}
```

Mod 执行前必须校验：
- `favor_delta` 按事件分档强制夹到安全范围。
- 正向 `favor_delta` 执行前读取 `NPCEx.GetFavor(npcId)`；当前好感度达到或超过 `60` 时归零，超过 `60` 的部分裁剪掉。
- 无明确情绪或关系变化时归零。
- 同一轮只允许一次好感度变化。
- 只有校验通过后才调用 `NPCEx.AddFavor(npcId, delta)`。
- 成功后显示简短游戏提示。
- 同时记录 NPC 长期记忆，影响后续口吻和关系阶段。

## 角色卡与 Prompt 权重
- 当前 Mod 优先读取运行目录角色卡：
  `C:\Users\22749\AppData\LocalLow\yusuiInc\觅长生\{npcId}_{npcName}_character.txt`
- 如果运行目录角色卡不存在，会尝试从插件目录 `RPGSkillKit/character_cards_minimal` 初始化一张角色卡。
- 带 NPC ID 的草稿卡按文件名前缀匹配，例如 `20288_ni_xuxin.character.txt`。
- 无 NPC ID 的特殊角色使用少量名称映射匹配，例如魏无极、星铃儿/星凝、吞云大圣、火麒麟、冲虚、白帝。
- 找不到初始卡时保持旧逻辑：创建 `空白角色卡`。
- 玩家已手动编辑过的运行目录角色卡优先级最高，不会被内置草稿覆盖。
- NPC 聊天 prompt 的最高优先块包含：角色卡、玩家与 NPC 关系定位、玩家境界、NPC 境界、境界礼法约束。
- 境界礼法约束用于避免低境界 NPC 对高境界前辈无意义猖狂；如果 NPC 无法确认玩家境界，应谨慎试探，不应默认冒犯。

## 对话触发战斗 V2
- 已有好感度分级不变：普通辱骂/威胁只扣好感，不直接开战。
- 当本轮效果为 `insult_attack / severe / -25`，且 NPC 回复明确表示开战、拔剑、动手、一战等意图时，Mod 会在好感度扣减后触发标准战斗。
- 战斗触发流程：NPC 回复 -> 系统提示“对方怒意已决，好感度 -25，即将进入战斗。” -> 调用 `StartFight.Do(...)`。
- NPC 只是警告、驱赶、表示不计较时，不触发战斗。
- 游戏已有战斗前整备环节，因此 V2 不增加二次确认按钮。
- `plugins/prompt.txt` 只提示 NPC 可以在正文中明确表达开战意图；AI 不能直接执行战斗命令，最终仍由 Mod 本地校验。

## 风险和边界
- 不应让 AI 直接执行游戏命令；AI 只能建议，Mod 必须裁决。
- 单轮关系变化要小，避免一句话破坏长期养成节奏。
- 战斗触发已进入第二阶段；当前只触发标准战斗，不改 NPC 死亡、任务状态或长期敌对状态。
- 如果未来需要提交 GitHub，二进制 DLL/PDB/XML 依赖应继续被 `.gitignore` 或选择性提交策略保护。
