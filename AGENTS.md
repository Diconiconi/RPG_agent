# RPG_agent 项目说明

本文件适用于 `E:\RPG_agent`。后续 Codex 或其他模型接手时，优先遵守这里的项目规则。

## 默认沟通与文档

- 默认用中文回复、写 worklog、写交接说明；代码标识、命令、报错、提交信息可保留英文。
- 回答尽量短，优先说结论、改了什么、如何验证；不要展开长篇背景，除非用户要求。
- 用户没有要求“完整对齐/同步所有文档/完整流程”时，不大范围更新旧设计文档和旧计划书。

## 当前项目方向

- 不另建独立 mod 工程，优先直接改 `参考mod/` 里的现有 AI 对话 mod。
- 重点代码在 `参考mod/src/MCS_AIChatMod/`，运行参考文件在 `参考mod/plugins/`。
- 当前机制方向：AI 只建议对话效果，Mod 本地校验后执行。
- 对话效果当前写入 NPC 好感度，不再写入其他关系数值通道。
- 通过聊天获得的正向好感度最高到 `60`；达到或超过 `60` 后不再通过聊天增加。负向对话效果仍可降低好感度。
- 战斗触发、任务分支、强剧情状态变化属于后续阶段，未明确要求时不要加入 V1。

## 默认工作模式

- 默认采用“轻量小改模式”：
  - 优先在现有 mod 和现有文件上做最小必要修改。
  - 尽量保持原逻辑和原结构，不重做架构，不新建复杂流程。
  - 只改核心代码、必要测试、必要 prompt 或当前交接文档。
  - 不因为普通小改动同步旧设计文档、旧计划书或大范围工作记录。
- 小型只读诊断、纯聊天、很小的文案说明，不需要 worklog/提交/推送。
- 只有关键任务、重要行为变更或用户明确要求时，才写 worklog。
- Superpowers 工作流只在用户明确要求、任务明显复杂、或确实需要设计/计划约束时使用；使用时也应尽量压缩流程，避免为小改动制造过重文档和步骤。
- 如果用户明确说“完整流程”或“全部对齐”，再同步旧设计文档、旧计划书、AGENTS 等长文档。
- 实现原则：尽可能保持原来的逻辑，在原 mod 上加入自己的功能，实现方法尽量简单；优先小改动，不重做架构，除非用户明确要求。

## Git 与 worklog

- 重要功能或会影响运行行为的修改：最终报告前写 `worklogs/YYYY-MM-DD-*.md`，默认中文。
- 默认不 `git push`。只有用户明确说“上传”“推送”“push”或要求发布/同步到远程时，才推送到 GitHub。
- 默认不为纯文档偏好、小说明、只读诊断创建提交；重要功能完成后可本地提交，但仍不自动推送。
- 提交前检查：
  - `git status --short`
  - `git remote -v`
  - 远程应为 `https://github.com/Diconiconi/RPG_agent.git`
- 只提交相关文本/源码；不要提交 API key、认证文件、存档、运行日志、本地游戏文件、复制来的游戏二进制。
- 如果 push 因认证、权限或网络失败，保留本地提交并报告准确原因。

## 安全边界

- 除非用户明确要求，不要修改真实游戏安装目录。
- 默认只读：
  - `F:\SteamLibrary\steamapps\common\觅长生`
  - `C:\Users\22749\AppData\LocalLow\yusuiInc\觅长生`
- `参考mod` 中的 DLL、PDB、XML、BIN 可本地保留，但默认不要提交到 GitHub。
